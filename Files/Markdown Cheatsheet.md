# Jupyter Markdown

This document is most helpful in a side-by-side display with an editor and in Preview window. 

A file with markdown is pure text with “markup” that indicates how the text should be presented. 

## Lists

#### Use dashes to get bullet points 

- point 1
- point 2 

#### Use numbers for a numbered list 

1. First step 
2. Second step 

## Math 

In Jupyter Lab, you can use markup from Latex to generate both inline math like this $f(x) = x^2$ or display math:  

$$ \sum_{n = 0}^\infty \frac{1}{n^2} $$

## Code 
Back ticks delimit a fragment of code: `print(x)`


## Dashes and Hyphens 

Hyphen -

> Use to break words (at the end of a line) or combine them:
>
> "mass-produced" 

n-dash or en dash: U+2013 

> Use to indicate a range:
>
> "The aughts refer to the years 2000–2009."  

m-dash or em dash: U+2014 

> Use to set off a distinct idea or clause:
>
> "I am going to assume&mdash;unless instructed otherwise—that we can proceed." 



## Curly Quotes

In code, the same straight double quotation-mark is used in code open and close a "quoted string". 

You can enter quotation marks with a look that is different for opening and closing using either the HTML for the quote or Unicode. Here is what they look like in this font: &ldquo;curly quotes&ldquo;

It is easier to enter HTML. See below for links that explain how to enter unicode from the keyboard. 

In English, curly quotes are given by: 

### HTML
> #### Double
> - open:  &ldquo
> - close: &rdquo
> #### Single
> - open: &lsquo
> - close &rdquo

### Unicode 
> #### Double
> - open:  &ldquoU+201C
> - close: U+201D
> #### Single
> - open: U+2018
> - close U+2019

More on unicode [here](https://unicode-table.com/en/sets/quotation-marks/).
